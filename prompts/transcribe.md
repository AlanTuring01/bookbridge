# 逐章转写 / 翻译提示词模板

把下面的占位符替换成你的实际内容，然后把「一章的页图 + 这章的 OCR 草稿」一起喂给
一个带视觉能力的 AI（Claude、GPT-4o 等）。它会对照原页图，把机器草稿修成
出版级的逐页 Markdown。一次处理一章（约 15–25 页），质量远胜整本一次性硬转。

---

你是资深图书编辑兼译者。{{授权声明：如转换/翻译受版权保护的作品，请在此说明你拥有版权或已获授权}}。

任务：把 {{书名}} 第 {{章名}}（源 PDF 第 {{起}}–{{止}} 页）的扫描页，逐字转写为规范
{{目标：简体中文 / 繁体中文 / 某语言}} 的 Markdown。

资料：
- 页图（唯一权威，逐页查看）：{{p021.png … p038.png 的路径}}
- OCR + 繁简转换的机器草稿（仅供加速，常漏字、有形近错字，以页图为准）：{{draft 路径}}

规则：
1. 每页以单独一行 `<!-- PAGE N -->` 开头（N 用源 PDF 的页序）。
2. 把 OCR 的断行合并成完整段落，保留作者原有分段；段落跨页时，后页接续的半段前加
   一行 `<!-- CONT -->`。
3. 删除页眉、页脚、页码、书眉等非正文元素，不要写进输出。
4. 标题按版面层级：用 `# / ## / ###`。原书的粗体用 `**加粗**`。
5. 竖排页（扉页题字、古籍等）按从右到左的列序阅读后再转写。
6. 字形与标点规范化：
   - 繁→简：甚麼→什么、裏/裡→里、後→后、瞭解→了解；引号「」→“”、『』→‘’。
   - 简→繁：什么→甚麼、里→裡/裏、了解→瞭解；引号“”→「」、‘’→『』。
   - 「著/着」「乾/干」等一字多形的，按上下文选对字。
7. 人名、专名规范化；外文、数字、公式照原样保留。
8. 照片 / 图表 / 信息图：在版面对应位置插入一行
   `<!-- FIGURE page=N type=照片|图表|信息图|插画 caption="图注" -->`；图内文字不必转成正文，
   但图题/图注要写进 caption。
9. 整页设计页（章扉页、部扉页）：输出 `<!-- DESIGN-PAGE page=N title="标题" -->`，下一行把
   扉页文字转写出来。纯色无内容衬页 / 真空白页：输出 `<!-- BLANK page=N -->`。
10. 表格转成 Markdown 表格；列表保留编号 / 项目符号。
11. **绝不概括、缩写、补写、跳页**——必须逐字转写全部正文。模糊难辨的字用 〔?〕标注。
12. 原书的排印错误**照原样保留**，只在交付说明里记录，不要擅自「改正」。

完成后：把整章按页序写入 `{{输出路径，如 work/ch01.md}}`，再用一段话说明：处理页数、
FIGURE/DESIGN/BLANK 的页码、主要修复点、以及发现的原书排印问题。
